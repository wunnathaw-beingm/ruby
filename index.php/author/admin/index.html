&とto_procメソッド

&について以前にも説明しましたが、Procオブジェクトをブロックとして渡したい場合は、引数の前に&を付ける必要があります。 &の役割はProcオブジェクトをブロックとして認識させて、右辺のオブジェクトに対してto_procメソッドを呼び出して、戻り値として得られたProcオブジェクトをブロックを利用するメソッドに与えます。 

sample_proc = Proc.new { |s| s.reverse }
['apple', 'orange', 'banana'].map(&sample_proc) 
#=>  ["elppa", "egnaro", "ananab"]

元々からProcオブジェクトだった場合はto_procメソッドを呼んでも自分自身を返るだけです。

sample_proc = Proc.new { |s| s.reverse }
other_proc = sample_proc.to_proc
other_proc.equal?(sample_proc)   #=> true

RubyにはProcオブジェクト以外でto_procメソッドをシンボルも持っています。次のように:splitというシンボルをto_procメソッドでProcオブジェクトに変換します。

split_proc = :split.to_proc
split_proc
#=>  #<Proc:0x0000000003665398(&:split)>
split_proc = :split.to_proc
#引数が1つの場合はホワイトスペースで分割する
split_proc.call('a-b-c-d e')   #=> ["a-b-c-d", "e"]

引数を2個渡すと、１つめの引数はレシーバのままですが、２つめの引数がシンボルで指定したメソッドの第1引数になります。

split_proc = :split.to_proc
split_proc.call('a-b-c-d e', '-')   #=> ["a", "b", "c", "d e"]

上記の通り引数が2の場合は’a-b-c-d e’.split(‘-‘)と同じく指定された文字で分割されます。引数が3個の場合は、３つ目の引数がシンボルで指定したメソッドの第2引数になります。

split_proc = :split.to_proc
split_proc.call('a-b-c-d e', '-', 3)   #=> ["a", "b", "c-d e"]

Procとラムダ

Procオブジェクトを作る方法は4つあります。Proc.newまたはprocメソッドでProcオブジェクトを作成する方法を説明しました。次のように構文やメソッドを使ってもProcオブジェクトを作成することができます。

# Procオブジェクトを生成
proc_sample = ->(a, b) { a + b }

# Procオブジェクトを生成
proc_sample2 = lambda { |a, b| a + b }

上記の引数のリストに使っている括弧()は省略できます。

proc_sample = -> a,b { a + b }

引数がなければ全て省略できます。

proc_sample = -> { 'Hello!' }

{}のかわりにdo…endを使うこともできます。

-> (a, b) do
  a + b
end

Proc.newと同じように、引数のデフォルト値を持つことができます。

proc_sample = ->(a = 0, b = 0) { a + b }

Proc.newとラムダの利用に異なる点が２点あります。
Proc.newもラムダも引数の扱いに違いがありません。以下はProc.newとラムダをそれぞれ実行するコードです。

#Proc.newの作成と実行
proc_sample = Proc.new { |a, b| a + b }
proc_sample.call(1, 2)   #=> 3

#ラムダの作成と実行
proc_sample2 = ->(a, b) { a + b }
proc_sample2.call(1, 2)  #=> 3

１つ目の異なる点はラムダを利用する場合、Proc.newよりも引数のチエックが厳密になります。

#Proc.newの場合、引数がnilでもエラーが発生しないようにto_iメソッドを使う
proc_sample = Proc.new { |a, b| a.to_i + b.to_i }
proc_sample.call(1)   #=> 1    #Procの引数は1つでも呼び出す可能
proc_sample.call(1, 2, 3)   #=> 3　　　#Procの引数は3つでも呼び出す可能

#ラムダ
proc_sample2 = ->(a, b) { a.to_i + b.to_i }
proc_sample.call(1) #=> ラムダは引数の個数に厳密なので、エラーになる。
#=> ArgumentError: wrong number of arguments(given 1, expected 2)
proc_sample.call(1, 2, 3)
#=> ArgumentError: wrong number of arguments(given 3, expected 2)

Proc.newもラムダもProcクラスのインスタンスです。ProcクラスのインスタンスがProc.newとして作られたのか、ラムダとして作られたのかをlamda?メソッドで判断できます。ラムダとして作られたオブジェクトであればtrueを返します。

#Proc.newの場合
proc_sample = Proc.new { |a, b| a + b }
proc_sample.lambda?    #=> false
proc_sample.class      #=> Proc

#ラムダの場合
proc_sample2 = ->(a, b) { a + b }
proc_sample2.lambda?  #=> true
proc_sample2.class    #=> Proc

■Proc.newとラムダのreturnとbreakの違い

２つ目の異なる点はProc.newやラムダの内部でreturnとbreakを使った場合挙動は異なります。次はreturnを使った場合の例です。

def proc_return
  proc_sample = Proc.new { |n| return n * 10 }
  result = [1, 2, 3].map(&proc_sample)
  "result: #{result}"
end

def lambda_return
  proc_sample = ->(n) { return n * 10 }
  result = [1, 2, 3].map(&proc_sample)
  "result: #{result}"
end

proc_return   #=> 10
lambda_return #=>  "result: [10, 20, 30]"

Proc.newではreturnを使うとメソッドを抜けます。一方でラムダの方はメソッドを抜けるのではなく、ラムダ内の処理から抜けるだけです。そのため、map処理を最後まで実行されます。次はbreakを使った場合の例です。

def proc_break
  proc_sample = Proc.new { |n| break n * 10 }
  result = [1, 2, 3].map(&proc_sample)
  "result: #{result}"
end

def lambda_break
  proc_sample = ->(n) { break n * 10 }
  result = [1, 2, 3].map(&proc_sample)
  "result: #{result}"
end

proc_break   #=> LocalJumpError (break from proc-closure)
lambda_break #=>  "result: [10, 20, 30]"

Proc.newの内部でbreakを使うと例外が発生します。一方でラムダを使う場合は例外が発生しないし処理を最後まで実行されます。

Procの使い方

Procクラスはブロックオブジェクト化するためのクラスです。stringクラスであれば文字列を、integerクラスであれば整数を表します。Procクラスはブロックを表します。Procクラスのインスタンスを作成するときにはProc.newにblockを渡します。 

hello_proc = Proc.new do
    'Hello!'
end
#do...endのかわりに{}を使えます。
hello_proc = Proc.new{'Hello!'}

Proc.newを使わずに別の方法としてKernelモジュールのproc methodも利用してインスタンスの作成もできます。

#proc.newのかわりにprocメソッドを利用する
add_proc = proc { |a, b| a + b }

Procオブジェクトはオブジェクトとして存在するだけでそのまま実行されません。実行させたい場合はcallメソッドを利用します。

hello_proc = Proc.new{'Hello!'}
hello_proc.call     #=> 'Hello!'

実行する時に引数を渡すこともできます。

add_proc = Proc.new{ |a, b| a + b}
add_proc.call(10, 20)   #=> 30

引数にデフォルト値を付けることもできます。

add_proc = Proc.new{ |a = 0, b = 0| a + b}
add_proc.call           #=> 0
add_proc.call(10)       #=> 10
add_proc.call(10, 20)   #=> 30

Procを実行させるためにはcallメソッド以外に他の方法でも実行できます。

add_proc = Proc.new { |a, b | a + b}

puts "use call method: #{add_proc.call(1, 2)}"
puts "use yield method: #{add_proc.yield(1, 2)}"
puts "use .() : #{add_proc.(1, 2)}"
puts "use [] : #{add_proc[1, 2]}"
puts "use === : #{add_proc === [1, 2]}"

#=> use call method: 3
#   use yield method: 3
#   use .() : 3
#   use [] : 3
#   use === : 3

■Procオブジェクトをブロックの代わり渡す

procオブジェクトはブロックと同じように処理のかたまりを表しますが、ブロックとは異なり、オブジェクトとして扱うことができます。変数に入れて別のメソッドに渡したり、procオブジェクトに対してメソッドを呼び出すことができます。

yieldで説明した通りブロックは&を利用して引数として受け取ることができます。引数のblockはprocオブジェクトであります。次のように出力してみたらわかります。

def hoge(&block)
    puts block.class
    text = block.call('Kentaro')
    puts text
end

hoge do |text|
    "hello, #{text}"
end

#=> Proc
#   hello, Kentaro

■Procオブジェクトを普通の引数として渡す

Procオブジェクトを下記のように引数として受け取ることができます。

def hoge(regular_proc)
    puts 'Hello!'
    text = regular_proc.call('How are you?')
    puts text
    puts 'Good morning!'
end

greeting_proc = Proc.new { | text | text * 2 }
hoge(greeting_proc)

#=> Hello!
#   How are you?How are you?
#   Good morning!

ここで注意することは、メソッドが受け取れるブロックの数は最大で1つですが、Procオブジェクトは引数として渡す分に制限はありません。下記の例では３つprocオブジェクトを引数として渡してます。

def hoge(proc_1, proc_2, proc_3)
    puts proc_1.call('Hello!')
    puts proc_2.call('This is Proc 2!!')
    puts proc_3.call('proc 3')
end

greeting_proc = Proc.new { | text | "#{text}, Michael!" }
repeat_proc = Proc.new { | text | text * 2 }
question_proc = Proc.new { | text | "Is this #{text} ?" }

hoge(greeting_proc, repeat_proc, question_proc)

#=> Hello!, Michael!
#   This is Proc 2!!This is Proc 2!!
#   Is this proc 3 ?

yieldの使い方

yieldはブロック付きメソッドを呼び出すことができます。
ここでブロックについて説明します。ブロックとはdo～endや{}で囲まれた処理のかたまりです。以下の例だと「puts i」部分はブロックになります。

(1..10).each do |i|
	puts i
end

メソッドを呼び出す方法は２種類あります。１番目はメソッド名を普通に呼び出して利用します。

def hoge
  puts "hello hoge"
end

hoge
#=> hello hoge

2番目はメソッドをブロック付きで呼び出してみます。

def hoge
  puts "hello hoge"
end

hoge do
  puts "hello hoge2"
end

#=> hello hoge

上記のコードを見ると、ブロック付きでメソッドを呼び出す時にエラーが発生しないです。出力の結果には変わりはないですが、ブロックのコードは実行されません。このような場合、ブロックのコードを実行させるためyieldを利用します。

def hoge
  puts "hello hoge"
  yield
end

hoge do
  puts "hello hoge2"
end

#=> hello hoge
#   hello hoge2

上記の例のとおり、yieldを利用することでブロックのコードも実行されました。 yieldを複数回書けば、ブロックも複数回呼び出されます。 

def hoge
  "Hoge1"
  yield
  yield
  "Hoge2"
end

hoge do
  puts "Hello"
end

#=> "Hoge1"
#=>	"Hello"
#=> "Hello"
#=> "Hoge2"

ブロックなしでyieldを利用した場合エラーが発生します。

def hoge
  yield
end

hoge

#=> LocalJumpError (no block given (yield))

ブロックが渡されたいたかblock_given?メソッドを利用して確認することができます。

def hoge
  puts "hoge"
  #ブロックが渡されたか確認
  if block_given?
    yield
  end
end

hoge do
  puts "Hello"
end

#=> "hoge"
#=> "Hello"

#ブロックなしメソッド

hoge
#=> "hoge" (今回はエラーが発生しない）

yieldはブロックに引数を渡したり、ブロックの戻り値を受け取りことができます。

def hoge
  text = yield "Kentaro"
  puts text
end

hoge do |text|
  "hello, #{text}"
end

#=> hello, Kentaro

■ブロックを引数として受け取る

ブロックを引数として受け取るためには、引数名の前に「&」を付ける必要があります。そのブロックをcallメソッドを使って実行できます。

def hoge(&block)
  text = block.call('Kentaro')
  puts text
end

hoge do |text|
  "hello, #{text}"
end

#=> hello, Kentaro

ブロックを引数にするメリットはブロックをほかのメソッドに引き渡せるようになることです。

def hoge(&block)
  introduction("Kentaro", &block)
end

def fuge(&block)
  introduction("Sakura", &block)
end

def introduction(text, &block)
  puts text
  puts block.call(text)
end

hoge do |text|
  "hello, #{text}"
end
#=> Kentaro
#=> hello, Kentaro

fuge do |text|
  "hello, #{text}"
end
#=> Sakura
#=> hello, Sakura

rescue修飾子

rescueを下記のように修飾子としても使えます。例外が発生した場合、rescue修飾子に書いた値が戻り値となります。

require 'date'

def to_date(string)
    begin
        Date.parse(string)
    rescue ArgumentError
        nil
    end
end

# 例外が発生しなかった場合
to_date('2021-05-19')
#=>  #<Date: 2021-05-19 ((2459354j,0s,0n),+0s,2299161j)>

# 例外が発生した場合
to_date('aaabbb')
#=> nil

being/end省略して下記のように書けます。

def to_date(string)
    Date.parse(string) rescue nil
end

# 例外が発生しなかった場合
to_date('2021-05-19')
#=>  #<Date: 2021-05-19 ((2459354j,0s,0n),+0s,2299161j)>

# 例外が発生した場合
to_date('aaabbb')
#=> nil

　上記の通りbegin/endを省略することで短く書くことができましたが、捕捉する例外クラスを指定することはできません。rescue修飾子を使うとStandardErrorとそのサブクラスが捕捉されます。例外処理を細かく制御したい場合はbegin/endを使って書く方が良いです。

例外処理の戻り値

例外処理にも戻り値があります。例外処理の戻り値は二種類あります。最後まで正常に進んだ場合begin節の最後の式が戻り値になります。例外が発生した場合は、rescue節の最後の式が戻り値として返します。

# 例外が発生しない場合、begin節の最後の式が戻り値
foo = 
    begin
        'Hello'
    rescue
        'error'
    ensure
        'ensure'
    end
foo #=> "Hello"
#例外が発生した場合、rescue節の最後の式が戻り値
bar = 
    begin
        1/0
    rescue
        'error'
    ensure
        'ensure'
    end
bar #=> "error"

例外処理のelse節

例外処理が発生しなかった場合にelse節を書いておくとelse節が実行されます。

begin
    # 例外処理が発生する可能性がある処理
rescue
    # 例外処理が発生した場合の処理
else
    # 例外処理が発生しなかった場合の処理
ensure
    # 例外の有無にかかわらず実行する処理
end

else節を使わなくても、begin節に例外が発生しなかった場合の処理を書いおくと良いので、else節が使われるケースは多くありません。

begin
    puts 'Hello World'
rescue
    puts 'exceptionが発生しました'
else
    puts 'exceptionが発生しませんでした'
end
#=> "Hello World"
#   "exceptionが発生しませんでした"

以下はelse節を使わない例です。出力されるのは同上です。

begin
    puts 'Hello World'
    puts 'exceptionが発生しませんでした'
rescue
    puts 'exceptionが発生しました'
end
#=> "Hello World"
#   "exceptionが発生しませんでした

retryやensureの使い方

■retry
例外発生時にもう一度処理をやり直したいときretryを使います。ネットワークエラーのように一時時に発生している例外であれば、何度かやり直すことで正常に実行できる可能性があります。その時はretry文を使います。retry文を実行すると、begin節の最初からやり直します。

begin
    # 例外が発生する可能性がある処理
rescue
    retry # 処理をやりなおす
end

retryを使う場合例外が解決しない時は無限ループを作ってしまう恐れがあるので、カウンタ変数を用いて、retryの回数を制限するのが良いです。

max_retry_count = 3
retry_count = 0
begin
    puts 'start process'
    1/0
rescue
    retry_count += 1
    if retry_count <= max_retry_count
        puts "retry します。#{retry_count}回目"
        retry
    else
        puts 'retryを終了しました。'
    end
end

#=> start process
#   retry します。1回目
#=> start process
#   retry します。2回目
#=> start process
#   retry します。3回目
#=> start process
#   retryを終了しました。

■entry
例外処理を入れた場合、例外が発生してもしなくても必ず実行したい処理がある場合にentryを利用します。

begin
    # 例外が発生る可能性がある処理
rescue
    # 例外が発生した時の実行処理
ensure
    # 例外が発生してもしなくても実行する処理
end

以下の例のように例外発生の有無にかかわらず、fileオブジェクトをクローズしたい場合に利用します。

# fileを開く
file = File.open('sampleFile.txt', 'w')
begin
    # fileに書き込む
    file << 'Hello World'
ensure
    # 例外の有無にかかわらず、fileを閉じる
    file.close
end

ensureの代わりにブロック付きでfileを開くと、メソッドの実行後にfileを自動的に閉じます。

File.open('sampleFile.txt', 'w') do |file|
    file << 'Hello'
end

例外処理クラスの継承関係

rescue節で例外クラスを指定する場合は、例外クラスの継承関係を理解しておく必要があります。ZeroDivisionErrorやNoMethodErorクラスはExceptionクラスを継承しています。Exceptionクラスのサブクラスとして、NoMemoryError,SystemExit、StandardError等が存在します。StandardErrorクラスは通常のプログラムで発生する可能性が高い例外を表すクラスです。StandardErrorにはサブクラスとしてRuntimeError、NameError、TypeError、ArgumentError等があります。rescue節にクラスを何も指定しなかった場合はStandardErrorとそのサブクラスです。NoMemoryError、SystemExitなどのStandardErrorを継承していない例外クラスを捕捉されません。

begin
    # 例外が起きる処理
rescue
    # StandardErrorとそのサブクラスのみ捕捉される
end

下記のようにExceptionクラスを指定すると、StandardErrorと無関係のExceptionクラスのサブクラス全てが捕捉されます。

begin
    # 例外が起きる処理
rescue Exception
    # Exceptionクラスとそのサブクラスが捕捉される。
end

通常のプログラムで捕捉するのはStandardErrorクラスか、そのサブクラスに限定すべきです。

begin
    # NoMethodErrorを発生させる
    'foo'.bar
rescue NameError
    # NoMethodErrorはここで捕捉される
    puts 'NameError'
rescue NoMethodError
    # このrescueは実行されない
    puts 'NoMethodError'
end
#=> NameError

2つ目のrescue節が実行されないのは、NameErrorはNoMethodErrorのスーパークラスなので、最初に指定したNameErrorクラスのrescueで捕捉されます。NameErrorとNoMethodクラスの関係は下記の通りです。

Exception > StandardError > NameError > NoMethodError

上記例の問題を解決するためにはスーパークラスよりサブクラスを手前に持ってくる必要があります。次の例をご覧ください。

begin
    'foo'.bar
rescue NoMethodError
    # NoMethodErrorはここで捕捉される。
    puts 'NoMethodError'
rescue NameError
    puts 'NameError'
end
#=> NoMethodError

上記の例の通りNameErrorのより先にNoMethodErrorで捕捉されるため、NoMethodError用の例外処理を実行することができます。NameErrorが発生した場合はNameError用の例外処理を実行することができます。

begin
    # NameErrorを発生させる
    Foo.new
rescue NoMethodError
    puts 'NoMethodError'
rescue NameError
    puts 'NameError'
end
#=> NameError

例外処理を作成する方法

例外処理を下記のように作成することができます。

begin
 #例外が起きうる処理
rescue => e
 #例外が発生した場合の処理
end

上記の例のeは例外オブジェクトを格納する変数です。例外オブジェクトを格納する変数の名前は自由に付けることができます。eやexのような名前をよく付けられています。オブジェクトのメソッドを呼び出すことで、発生した例外に関する情報を取得することができます。messageメソッドは例外発生時のエラーメッセージを返してくれます。backtraceメソッドはバックトレースを配列として返してくれます。具体的例は以下の通りです。

begin
  1/0
rescue => e
  puts "error class: #{e.class}"
  puts "error message: #{e.message}"
  puts "backtrace....."
  puts e.backtrace
  puts "............"
end


#結果
#=> error class: ZeroDivisionError
#   error message: divided by 0
#   backtrace.......
#   (irb):11:in `/'
#   (irb):11:in `rescue in irb_binding'
#   (irb):10:in `irb_binding'
#   C:/Ruby26-x64/bin/irb.cmd:31:in `<main>'
#   .................

エラーが発生する下記のコードがあります。

begin 
    1 / 0
end

上記のコードを実行をするとZeroDivisionErrorが発生します・。

ZeroDivisionError (divided by 0)

そのエラーが発生した場合、rescue節のコードが実行されます。

begin 
    1 / 0
rescue  ZeroDivisionError
  puts "0で除算しました"
end

#=> 0で除算しました
