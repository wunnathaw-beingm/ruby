メソッド探索のルール

Rubyにおけるメソッド探索のルールをhelloメソッドを題材として取り上げます。

module ModuleA
  def to_s
    "ModuleA #{super}"
  end
end

module ModuleB
  def to_s
    "ModuleB #{super}"
  end
end

class ClassA
  def to_s
    "ClassA #{super}"
  end
end

class ClassB < ClassA
  include ModuleA
  include ModuleB
  def to_s
    "ClassB #{super}"
  end
end

obj = ClassB.new
obj.to_s  #=> "ClassA ModuleB ModuleA ClassA #<ClassB: 0x0000000035237c8>"

 ClassB.ancestors  #=> [ClassB, ModuleB, ModuleA, ClassA, Object, Kernel, BasciObject]

上記の出力結果をみると次のような順番でto_sメソッドが呼び出されます。
・ClassB自身のto_sメソッド
・2番目にincludeしたモジュールBのto_sメソッド
・1番目にincludeしたモジュールAのto_sメソッド
・スーパークラスのClassAのto_sメソッド
・ClassAのスーパークラスのObjectクラスのto_sメソッド

呼び出される順をancestorsメソッドを呼び出せて、クラスやモジュールがどの順番でメソッド探索されるか確認することができます。もし最後のBasicObjectクラスでもメソッドが見つからなかった場合はNoMethodErrorが発生します。

状態を保持するモジュールの作成

クラスインスタンス変数を使って、クラス自身にデータを保持するができるように、モジュールでも状態を保持することができます。 

module TestApi
    @url = ''
    @debug_mode = false
    
    class << self
      attr_accessor :url, :debug_mode  
    end
end

TestApi.url = "http://example.com"
TestApi.debug_mode = true

puts TestApi.url            #=> "http://example.com"
puts TestApi.debug_mode     #=> true

上記の例もモジュールではなくクラスを使って実現することができます。モジュールはインスタンス化できない点がクラスと異なります。インスタンスを作って操作する必要がないのであれば、モジュールにしておいたほうがいいです。

関数や定数をまとめるモジュール

モジュールをクラスにミクスインする場合includeやextendを使うことができます。クラスに組み込まなくてもモジュール単体でそのメソッドを呼ぶ出すこともできます。こういう場合はモジュール自身に特異メソッドを定義して、”モジュール名.メソッド名”という形で呼ぶ出すことができます。 

module FirstModule
    def self.foo
        puts "foo"
    end
end

#他のクラスにミックスインしなくてもモジュール単体でそのメソッドを呼び出せる
FirstModule.foo       #=> foo

モジュールはクラスと違ってインスタンスが作れないため、newする必要がないです。モジュールでもクラスと同様にclass<<selfを使って、特異メソッドを定義することができます。特異メソッドをたくさん定義する場合はメソッド名の前のself.がなくせる分、タイプ量を減らすことができます。 

module FirstModule
    class << self
        def foo
            puts "foo"
        end
       #以下他の特異メソッドを定義
    end
end

module_functionメソッドも利用することができます。 

module FirstModule
    def foo
        puts "foo"
    end
    
    module_function :foo
end

FirstModule.foo      #=> foo

module_functionメソッドを引数なしで呼び出した場合は、そこから下に定義されたメソッドがすべてモジュール関数になります。

module FirstModule
    #ここから下のメソッドはすべてモジュール関数
    module_function
    def foo
        puts "foo"
    end
    
    def bar
        puts "bar"
    end
end

FirstModule.foo     #=> foo
FirstModule.bar     #=> bar

クラスに定義を定義できたように、モジュールにも定数を定義することができます。定義のしかたや参照のしかたはクラスの場合と同じです。 

module TEST
   #定数を定義する
    TOWN = "TOKYO".freeze
 end
  
puts TEST::TOWN     #=> "TOKYO"

名前空間の作成

名前空間とはクラス名の重複を防ぐため利用されます。大規模なプロジェックになるとクラス名やメソッド名の重複が問題になることがあります。 

class Foo
    def bar
        puts "first bar method"
    end
end

class Foo
    def bar
        puts "second bar method"
    end
end

Foo.new.bar  #=> second bar method

上記の例だとクラス名とメソッド名は重複になっています。Rubyでは同じクラス名やメソッド名があってもエラーになりません。この場合、どうやって区別をしたらいいでしょう？この場合登場するのが名前空間です。 

module FirstModule
    class Foo
        def bar
            puts "first bar method"
        end
    end
end

module SecondModule
    class Foo
        def bar
            puts "second bar method"
        end
    end
end
#namespaceを使う場合、"module名::クラス名"のような形でクラス、メソッドを呼ぼ出すことができます。
FirstModule::Foo.new.bar  #=> first bar method
SecondModule::Foo.new.bar  #=> second bar method

名前空間として使うモジュールがすでにどこかで定義されている場合は、モジュール構文やクラス構文をネストさせなくても”モジュール名::クラス名”のような形でクラスを定義することもできます。 

module FirstModule
end

module SecondModule
    class Foo
        def bar
            puts "second bar method"
        end
    end
end

#モジュール名::クラス名の形式
class FirstModule::Foo
    def bar
        puts "first bar method"
    end
end

FirstModule::Foo.new.bar   #=> first bar method
SecondModule::Foo.new.bar  #=> second bar method

Kernelモジュール

Kernelモジュールのメソッドはどのクラスからでも参照できるメソッドです。ObjectクラスにKernelモジュールをincludeしているからKernelモジュールのメソッドをどこからでも使えるようになっています。次のメソッドはKernelモジュールが提供しているメソッドです。

  • puts
  • p
  • print
  • require
  • loop
class User
end

#UserクラスのクラスはClass
User.class    #=> Class

#Class のsuper classは Module
Class.superclass   #=> Module

#Module のsuper class はObject
Module.superclass  #=> Object

#Objectにて Kernel moduleをincludeしている
Object.include?(Kernel)     #=> true

Kernelモジュールのメソッドがどこからでも使えるようになっているのはObjectクラスがKernelモージュールをincludeしているからです。

Kernelモージュールが提供しているメソッドを下記の公式サイトで確認できます。

https://docs.ruby-lang.org/en/master/Kernel.html
https://docs.ruby-lang.org/ja/latest/class/Kernel.html

Comparableモジュール

Comparableモジュールは比較演算を可能にするモジュールです。Comparableモジュールをincludeすると次のメソッドが使えるようになります。 <、<=、==、>、>=、between?

Comparableモジュールを使うためにはinclude先のクラスで<=>演算子を実装しておく必要があリます。その演算子をUFO演算子とも呼ばれています。
a <=> bは次のような結果を返します。

1. a > b なら　整数
2. a==b なら 0
3. a < b なら　の負の整数
4.比較できない場合 nil

次の例のようにComparableモジュールをincludeして使えます。 

class Tempo
    include Comparable

    attr_reader :bpm

    #bpmはbeats per minuteの略
    def initialize(bpm)
        @bpm = bpm
    end

    def <=>(other)
        if other.is_a?(Tempo)
            #bpm同士を <=> で比較する
            bpm <=> other.bpm
        else
            #比較できない場合は nilを返す
            nil
        end
    end

    def inpect
        "#{bpm}bpm"
    end
end

t_120 = Tempo.new(120)      #=> 120bpm
t_180 = Tempo.new(180)      #=> 120bpm

t_120 > t_180       #=> false
t_120 <= t_180      #=> true
t_120 == t_180      #=> false

Enumerableモジュール

Enumerable モジュールは配列、ハッシュ、範囲等の繰り返し処理ができるクラスにincludeされているモジュールです。Enumerableモジュールがincludeされているかクラスに対し、include?メソッドを利用して確認できます。 

Array.include?(Enumerable)      #=> true
Hash.include?(Enumerable)       #=> true
Range.include?(Enumerable)      #=> true

Enumerable モジュールにはmap, select, find, count等のメソッドが定義されています。 Enumerable モジュール をincludeされているクラスであれば、そのメソッドを呼び出すことができます。 

#Array, hash, rangeでmapをメソッドを使う
[1, 2, 3].map { |n| n * 10 }                     #=> [10, 20, 30]
{ a: 1, b: 2, c: 3 }.map { |k, v| [k, v * 10] }  #=> [[:a, 10], [:b, 20], [:c, 30]]
(1..3).map { |n| n * 10 }                        #=> [10, 20, 30]
#all method
[1,2,3].all? { |v| v > 0 }      #=> true

[1,2,3].all? { |v| v > 1 }      #=> false
#any method
[1,2,3].any? { |v| v == 2 }     #=> true
# collect method
[1,2,3].collect { |v| v * 2 }   #=> [2, 4, 6]

モジュールのミックスイン

モジュールをクラスにincludeして機能を追加することを ミックスイン と言います。

module Loggable
  def log(text)
    puts "[LOG] #{text}"
  end
end

class Product
  #moduleをincludeする
  include Loggable
  
  def name
    log 'name is called'
    'MOVIE'
  end
end

product = Product.new
product.name  #=> [LOG] name is called
              #=> MOVIE

■モジュールに他のモジュールをincludeする

includeはクラスだけでなく、モジュールの中でも呼び出すことができます。あるモジュールをincludeすれば、そのモジュールがincludeしているモジュールもincludeしたことになります。

module Foo
  def foo
    "Foo"
  end
end

module Bar
  #別のモジュールをincludeする
  def bar
    "Bar"
  end
end

class C
  #Barモジュールをincludeする
  include Bar
end

c = C.new
c.bar   #=> "Bar"
c.foo   #=> "Foo"

モジュールの定義

Rubyのモジュールはclassと同じようにmodule内に関数の定義やプログラミング上での役割や振る舞いをまとめることができます。次のようにモジュールを定義することができます。

#helloメソッドを持つGreeterモジュールを定義
module Greeter
  def hello
    'hello'
  end
end

上記の例をみるとクラスの定義と似ていますが、モジュールはクラスと違ってモジュールからインスタンスを作成することができません。他のモジュールやクラスを継承することもできないです。

#helloメソッドを持つGreeterモジュールを定義
module Greeter
  def hello
    'hello'
  end
end

#モジュールのインスタンスは作成できない
greeter = Greeter.new    #=> NoMethodError

#他のモジュールを継承して新しいモジュールを作ることができない
module AwesomeGreeter < Greeter
end
#=> SyntaxError: syntax error, unexpected '<'

モンキーパッチについて

モンキーパッチ(monkey patch)とは、クラスやライブラリ等の既存メソッドを上書きするして、自分が期待する挙動に変更することです。以下はモンキーパッチをあてる例です。

#Userクラスは外部ライブラリで定義されている想定
class User
  def initialize(name)
    @name = name
  end
  
  def hello
    "Hello, #{@name}!"
  end
end

#モンキーパッチをあてる前の挙動
user = User.new('John')
user.hello    #=> "Hello, John!"

#helloメソッドにモンキーパッチをあてて独自の挙動を持たせる
class User
  def hello
    "#{@name}、Nice to meet you!"
  end
end

user.hello    #=>  "John, Nice to meet you!"

既存のメソッドをエイリアスメソッドとして残し、上書きしたメソッドの中で既存のメソッドを再利用することもできます。

#Userクラスは外部ライブラリで定義されている想定
class User
  def initialize(name)
    @name = name
  end
  
  def hello
    "Hello, #{@name}!"
  end
end

#モンキーパッチ
class User
  #既存のhelloメソッドをhello_originalとして呼び出せるようにする
  alias hello_original hello
  
  def hello
    "#{hello_original}, WelCome!"
  end
end

user = User.new('John')
user.hello    #=> "Hello, John!, Welcome!"
