オブジェクト、インスタンス、レシーバー

クラスは設計図です。車の設計図から赤い車や黒い車が作られると同様にオブジェクト指向プログラミングではクラスからさまざまなオブジェクトが作成できます。作られたオブジェクトは同じ属性やメソッドを持っていますが、オブジェクトごとに属性の中に保存されるデータは異なります。

#Alice Ruby、20歳というユーザーオブジェクトを作成する
alice = User.new('Alice', 'Ruby', 20)
#BoB Python、30歳というユーザーオブジェクトを作成する
bob = User.new('Bob', 'Python', 30)

#両方のオブジェクトがfull_nameメソッドを持っているが、中身は異なる。
alice.full_name     #=> Alice Ruby
bob.full_name       #=> Bob Python

また、オブジェクトとメソッドとの関係のオブジェクトのことをレシーバーと呼びます。

user = User.new('Alice', 'Ruby', 20)
user.full_name

上記のコードの例を見ると、userオブジェクトがfull_nameメソッドを呼び出しています。ここでのfull_nameメソッドのレシーバーがuserになります。

full_nameのことをメソッドと呼びますが、他のプログラミング言語では「関数」とか「サブルーチン」と呼びます。Rubyでは、「関数」とか「サブルーチン」とは呼びません。

上記の2行目のコードのfull_nameを「メッセージ」と言う用語を使って説明する場合もあります。

レシーバーとメッセージという呼び方あSmalltalkというオブジェクト指向言語で使われている呼び方です。RubyもSmalltalkの影響を受けている部分もあるので「レシーバー」と「メッセージ」という用語を使われる場合もあります。

superの使い方

superを使うと親クラスにある同名メソッドを呼び出すことができます。

class Parent
  attr_reader :name, :age
  
  def initialize(name, age)
    @name = name
    @age = age
  end
end

class Child < Parent
  attr_reader :grade
  
  def initialize(name, age, grade)
    super(name, age)
    @grade = grade
  end
end

john = Child.new("John", 18, 10)
john.name  #=> "John"
john.age   #=> 18
john.grade #=> 10

スーパークラスとサブクラスで引数の数が同じだった場合、引数なしのsuperを呼ぶだけでスーパークラスに引き渡すことができます。また、スーパークラスとサブクラスで実行する処理が変わらなければ、サブクラスで同名メソッドを定義したりsuperを呼び出すしたりする必要がありません。

class Parent
  attr_reader :name, :age
  
  def initialize(name, age)
    @name = name
    @age = age
  end
end

 #スーパークラスとサブクラスで引数の数が同じだった場合
class Child < Parent
  def initialize(name, age)
    super
  end
end

john = Child.new("John", 18)
john.name  #=> "John"
john.age   #=> 18


#特別処理を実行ない場合は、同名メソッドやsuperを呼び出す必要がない
class Child < Product
end

john = Child.new("John", 18)
john.name  #=> "John"
john.age   #=> 18

initializeメソッドでsuperの使い方の例を記述してきましたが、initializeメソッド以外でも同じようにsuperを使うことができます。

class Car
  def accelerate
    print("acceleration!\n")
  end
end

class RacerCar < Car
  def accelerate
    super
    print("speed up acceleration!")
  end
end

racer = RacerCar.new
racer.accelerate

 #=>acceleration!
 #=>speed up acceleration!

クラスを継承する方法

クラスを定義するとデフォルトでObjectクラスを継承しています。

class Product
end

Objectクラス以外を継承する場合は<を使用して次のように書きます。

class Product
end

#DVDクラスがProductクラスを継承する
class DVD < Product
end

オブジェクトのクラスを調べる場合は、class_methodを利用します。

user = User.new
user.class      #=> User

user = User.new
#userはUserクラスのインスタンスかを確認

#userはStringクラスのインスタンスか確認

user = User.new

user.instance_of?(Object)   #=> false

#is_a?メソッドはis-a関係であればtrueになる
user.is_a?(User)            #=> true
user.is_a?(object)          #=> true
user.is_a?(BasicObject)     #=> true

#is-a関係ではないならfalse
user.is_a?(String)        #=> false

selfキーワードの使い方

Rubyのselfとはインスタンス自身を表すキーワードです。
selfはクラス内部で書かれる場合、そのクラスのインスタンス変数の参照に利用されます。

class User
  attr_accessor :name
  
  def initialize(name)
    self.name = name
  end
  
  def my_name 
    "my name is #{name}."
  end
  
  def greet
    "Hello, I'm  #{self.name}"
  end
  
end

john = User.new("John")
john.my_name    #=> "my name is John"
john.greet      #=> "Hello, I'm John"

また、selfをメソッドに付与するとインスタンスメソッドではなく、クラスメソッドとして定義することもできます。

class User
  attr_accessor :name
  
  def initialize(name)
    self.name = name
  end
  
  def self.my_name
    "This is class Method"
  end
  
  def my_name
    "My name is #{name}"
  end
end

john = User.new("John")
john.my_name    #=> "My name is John"
User.my_name    #=> "This is class Method"

selfの付け忘れで不具合が発生するケースもありますので、selfの使いには注意してください。

class User
  attr_accessor :name
  
  def initialize(name)
    @name = name
  end
  
  def rename_to_John
    self.name = "John"
  end
  
  def rename_to_Micheal
    #selfの付け忘れ
    name = "Micheal"     
  end
end

foo = User.new("Foo")
foo.rename_to_John
foo.name   #=> "John"
foo.rename_to_Micheal     #Michealにリネームできていない
foo.name  #=> "John"

クラスメソッドの公開レベル

Rubyのメソッドには以下のような3つの公開レベルがあります。

  • public
  • protected
  • private

■publicメソッド

publicメソッドはクラスの外部から呼び出せるメソッドです。initializeメソッド以外はデフォルトでpublicメソッドになります。

class User
  def greeting
    'Hello!'
  end
end

user = User.new
user.greeting  #=> 'Hello!'

■privateメソッド

privateメソッドはクラスの外部から呼び出すことができず、クラス内部でのみ使えるメソッドです。クラス内でprivateキーワードを書くと、そこから下で定義されたメソッドはprivateメソッドになります。

class User
  #ここから下で定義されるメソッドはprivate
  private
  
  def greeting
    'Hello!'
  end
end

user = User.new
user.greeting  #=> NoMethodError

privateメソッドはレシーバを指定して呼び出すことができないメソッドです。

class User
  #nameメソッドはprivateなので、レシーバselfを指定して呼び出すとエラーが発生する
  def greeting
    "Hello, I am #{self.name}."
  end
  
  private
  
  def name
    'John'
  end
end

user = User.new
user.greeting   #=> NoMethodError: private method 'name' called for <User>

privateメソッドはサブクラスでも呼び出せます。

class Animal
  private
  def species
    "Animal Species"
  end
end

class Dog < Animal
	def type
      #speciesはスーパークラスのprivateメソッド
      "type: #{species}"
    end
end

pappy = Dog.new
pappy.type   #=> "type: Animal Species"

スーパークラスのprivateメソッドが呼び出すことができるので、オーバーライドもできます。

class Animal
   
  def animalType
    "type: #{species}"
  end
  
  private
  
  def species
    "Animal Species"
  end
end

class Dog < Animal
  
  private
 #スーパークラスのprivateメソッドをオーバーライドする
  def species
    "Mammal"
  end
end

pappy = Dog.new
pappy.animalType   #=> "type: Mammal"

■protectedメソッド

protectedメソッドはそのメソッドを定義したクラス自身とそのサブクラスのインスタンスメソッドからレシーバ付きで呼び出せます。

class User
  attr_reader :name
  
  def initialize(name, weight)
    @name = name
    @weight = weight
  end
  
  def heavier_than?(other_user)
    other_user.weight < @weight
  end
  
  protected
  
  def weight
    @weight
  end
  
end

#同じクラス内のインスタンスメソッド内であれば、protectedメソッドが呼び出せる

john = User.new('John', 50)
micheal = User.new('Micheal', 60)

john.heavier_than?(micheal)    #=> false

#クラス外からはprotectedメソッドを呼び出せない

john.weihght    #=> NoMethodError

クラスメソッド

クラスメソッドを定義する方法は、メソッド名の前にself.を付けて定義する方法と、class << selfからendの間にメソッドを定義する方法として2パータンあります。

#クラスメソッド定義方法　その1
class User
  #クラスメソッド定義
  def self.my_name
    "my name is John"
  end
end

#クラスメソッド定義する方法　その2
class User
  class << self
    def my_name
      "my name is John"
    end
    
    def my_age
      "I am 18years old"
    end
  end
end

クラス構文の直下でクラスメソッドを呼ぶ出すことができます。

class User
  #クラスメソッド定義
  def self.my_name
    "my name is John"
  end
  
  def self.my_age
 	"I am 18years old"
  end
  
  self.my_name    #=> "my name is John"  
end

クラスメソッドをインスタンスメソッド内から呼び出す時は次のように書きます。

class User
  #クラスメソッド定義
  def self.my_name
    "John"
  end
  
  #インスタンスメソッド
  def name
 	#クラスメソッドを呼び出す
    "My name is #{User.my_name}."    #=> "My name is John."
  end
end

クラスを継承するとクラスメソッドも継承されます。

class Parent
  #クラスメソッド定義
  def self.parent_name
    "John&Cherry"
  end
end

class Child < Parent
end

#Parentを継承したChildからParentクラスのparent_nameメソッドを呼び出せる
Child.parent_name    #=>  "John&Cherry"

アクセサメソッド

Rubyにおけるアクセサメソッドとはインスタンス変数の値を読み書きするメソッドのことです。アクセサメソッドには3つの種類があります。

attr_reader    #読み取り用
attr_writer	   #書き込み用
attr_accessor  #読み書き用

■attr_reader

インスタンス変数の内容を読み取り専用にしたい場合は、attr_readerメソッドを使います。

class User
  attr_reader :name
  
  def initialize(name)
    @name = name
  end
end
  
user = User.new('John')
#@nameを参照できる  
puts user.name   #=> 'John'

#@nameを変更しようとするとエラーが発生する
user.name = 'Micheal'
#=> NoMethodError:  undefined method `name` for #<User:00x00000000034c1f28 @name='John'>

■attr_writer

インスタンス変数を書き込み専用にしたい場合は、attr_writerメソッドを使います。

class User
  attr_writer :name
  
  def initialize(name)
    @name = name
  end
end
  
user = User.new('John')
#@nameは変更できる
user.name = 'Micheal' 
#@nameを参照できない
puts user.name
#=> NoMethodError:  undefined method `name` for #<User:00x00000000034c1f28 @name='John'>

■attr_accessor

インスタンス変数の内容を読み書きしたい場合は、attr_accessorメソッドを使います。

class User
  attr_accessor :name
  
  def initialize(name)
    @name = name
  end
end
  
user = User.new('John')
#@nameは変更できる
user.name = 'Micheal' 

#@nameを参照できる 
puts user.name　　　　　#=> Micheal

クラスの変数について

Rubyにおける変数は下記のようにあります。

  1. ローカル変数
  2. インスタンス変数
  3. クラスインスタンス変数
  4. クラス変数
  5. グローバル変数
  6. 定数

■ローカル変数

ローカル変数についてはRubyに関する基礎知識にて紹介しましたので、ここでは割愛させていただきます。

■インスタンス変数

クラスの内部にインスタンス変数を使えます。インスタンス変数を同じオブジェクト、同じインスタンスからアクセスして利用できます。

class User
  def initialize(name)
    #インスタンス変数に引数のnameを代入する。
    @name = name
  end
  
  def hello
    #インスタンス変数を呼び出す
    "Hello, I am #{@name}."
  end
end

user = User.new('Alice')
user.hello       #=> "Hello, I am Alice."

■クラスインスタンス変数

インスタンス変数は@から始まる変数です。クラスインスタンス変数はクラス構文の直下や、クラスメソッドの内部で＠で始まる変数のことです。以下の例をご覧ください。

class Product
  #クラスインスタンス変数
  @name = 'Product'
  
  def self.name
    #クラスインスタンス変数
    @name
  end
  
  def initialize(name)
    #インスタンス変数
    @name = name
  end
  
  def name
    #インスタンス変数
    @name
  end
end

product = Product.new('movie')
product.name      #=>  movie
Product.name      #=>  Product

クラスインスタンス変数であるため、インスタンス作成と無関係に、Product.nameとしてアクセスすることができます。

■クラス変数

クラス変数はクラス内でもインスタンスメソッド内でも共有され、スーパークラスとサブクラスでもアクセスできます。クラス変数は@を2つ重ねて書きます。

class Product
  
  @@name = 'Product'
  
  def self.name
    @@name
  end
  
  def initialize(name)
    @@name = name
  end
  
  def name
    @@name
  end
end

class DVD < Product
  @@name = 'DVD'
  
  def self.name
    @@name
  end
  
  def upcase_name
    @@name.upcase
  end
end

#DVDクラスを定義したタイミングで@@nameが"DVD"に変更される
Product.name     #=>   "DVD"
DVD.name         #=>   "DVD"

product = Product.new('great movie')
product.name     #=> "great movie"

#Product.newのタイミングで@@nameが"great movie"に変更される
Product.name   #=> great movie
DVD.name       #=> great movie

dvd = DVD.new('awesome movie')
dvd.name      #=> "awesome movie"

#DVD.newのタイミングで@@nameが"awesome movie"に変更される
Product.name   #=> awesome movie
DVD.name       #=> awesome movie

■グローバル変数

グローバル変数は$で始まる変数です。クラスの内部、外部を問わず、プログラムのどこからでも参照できます。

$program_name = 'Awesome program'

class Program
    def initialize(name)
        $program_name = name
    end

    def self.name
        $program_name
    end

    def name
        $program_name
    end
end

#初期値が表示される
Program.name    #=> "Awesome program"

program = Program.new('Super Program')
program.name     #=>  "Super Program"

#Program.newのタイミングで$program_nameが"Super Program#"に変更される
Program.name     #=>  "Super Program"
$program_name    #=>  "Super Program"

■定数

クラス内に定数を定義することができます。

class Product
    DEFAULT_PRICE = 0

    attr_reader :name, :price

    def initialize(name, price = DEFAULT_PRICE)
        @name = name
        @price = price
    end
end

product = Product.new('A free movie')
product.price       #=> 0

定数は必ず大文字で始める必要があります。定数をクラスの外部から参照する場合は次のように書きます。

class Product
  DEFAULT_PRICE  = 0
end

Product::DEFAULT_PRICE     #=> 0

定数をクラスの外部から参照させたくない場合は、private_constantで定義する必要があります。

class Product
  DEFAULT_PRICE = 0
  private_constant :DEFAULT_PRICE
end

Product::DEFAULT_PRICE      #=> NameError

クラスの定数の値は書き換えることができます。値を書き換えることができますが、警告が表示されます。

class Product
    DEFAULT_PRICE = 0

    #定数値を書き換える
    DEFAULT_PRICE = 1000
end
#=> warning: already initialized constant Product::DEFAULT_PRICE
Product::DEFAULT_PRICE  #=> 1000

クラスの外部からも定数の値を書き換えることができます。

class Product
    DEFAULT_PRICE = 0
end

#クラスの外部から定数を書き換える
Product::DEFAULT_PRICE = 3000

#=> warning: already initialized constant Product::DEFAULT_PRICE
Product::DEFAULT_PRICE  #=> 3000

また、定数の値を再代入しなくてもミュータブルなオブジェクトであれば定数の値を変えることができます。ミュータブルなオブジェクトは配列、文字列、ハッシュ等です。

class Product
    SOME_NAMES = ['Foo', 'bar', 'Baz']

    def self.names_without_foo(names = SOME_NAMES)
        names.delete('Foo')
        names
    end
end

Product.names_without_foo   #=> ["Bar", "Baz"]

#定数値が書き換えられた
Product.SOME_NAMES          #=> ["Bar", "Baz"]

クラスの定義

クラスは次のように定義できます。クラス名は必ず大文字で始める必要があります。

class User
end

クラスからオブジェクトを作成する場合はnewメソッドを使います。

class User
end
User.new

User.newとしてオブジェクトを作成する時に呼ばれるのはinitializeメソッドです。他のプログラミング言語のコンストラクタのようなものと考えるとわかりやすいと思います。

class User
    def initialize
        puts 'Initialize'
    end
end
User.new      #=> Initialize

initializeメソッドは特殊なメソッドで、外部から呼び出すことはできません。

user = User.new
user.initialize
#=> NoMethodError (private method `initialize' called for #<User:0x000000000356dc88>)

case分と正規表現

正規表現はcase文のwhen節で使うことができます。case節で指定した文字列がwhen節で指定した正規表現にマッチするとwhen節の処理が実行されます。

text = '03-1234-5678'

case text
  when /^\d{3}-\d{4}$/
  	puts 'post number'
  when /^\d{4}\/\d{1,2}\/d{1,2}$/
  	puts 'date'
  when /^\d+-\d+-\d+$/
    puts 'phone number'
end

#=>  phone number
