等値を判断するメソッド

Rubyでは等値を判断するための equal ?とeql?メソッドがあります。

■equal?

equal?メソッドは同じインスタンスかどうかを判断する場合に使います。

a = 'abc'
b = 'abc'
a.equal?(b)         #=> false

c = a
a.equal?(c)         #=> true

■eql?

eql?メソッドは文字列の内容が等しいか判断する場合に使います。ハッシュで二つのキーが等しいかどうかを判定するのに使われます。

#文字列内容が等しいか判定する
a = 'abc'
b = 'abc'
a.eql?(b)        #=> true

#ハッシュの二つのキーが等しいか判定する
h = { 1 => 'Integer', 1.0 => 'Float' }
h[1]    #=>  "Integer"
h[1.0]  #=>  "Float"

#異なるキーとして扱われる
1.eql?(1.0)  #=> false

メソッド名の疑問符「?」と感嘆符「！」

■疑問符「?」

?で終わるメソッドは真偽値を返すメソッドを表します。

#空文字であればtrue、そうでなければfalse
''.empty?   #=> true
'abcd'.empty?    #=> false

#オブジェクトがnilであればtrue, そうでなければfalse
@foo.nil?     #=> true
@foo = 'foo'
@foo.nil?     #=> false

#奇数
1.odd?    #=> true

#偶数
2.even?     #=> true

■感嘆符「!」

!で終わるメソッドは破壊的メソッドを表します。メソッドを呼び出したオブジェクト自身が影響を受けることです。

array = [3, 1, 5, 2, 4]

a = array.sort
puts a   #=> [1, 2, 3, 4, 5]
puts array   #=> [3, 1, 5, 2, 4]

b = array.sort!
puts b   #=> [1, 2, 3, 4, 5]
puts array  #=> [1, 2, 3, 4, 5]

メソッドの定義と呼び出す方法

Rubyのメソッドの定義はdefから始まりendまでとなります。

def メソッド名(引数1, 引数2)
  #実行する処理
end

具体的な例は次のとおりです

def sum(num1, num2)
  num1 + num2
end
sum(1, 2)     #=> 3

メソッド名も変数名と同じルールになっています。1文字目は英文字かアンスコ( _ )で構成します。英文字以外をメソッド名に使えます。

#英文字で始める
def hello
  'Hello!'
end

#英文字とアンスコ
def say_hello
  'Hello!'
end

#メソッド名に数字を入れる
def say_hello_2
  'Hello!'
end

#数字から始まるメソッド名
def 2_say_hello
  'Hello!'
end                #=>SyntaxError

#メソッド名をひらがなや漢字でする
def 挨拶する
  'こんにちは！'
end
挨拶する  #=> "こんにちは！"

■メソッドの返り値について

returnを使用してメソッドの結果を変数に格納できます。

def sum(num1, num2)
  result = 0
  (num1..num2).each do |num|
    result += num
  end
  return result
end

#メソッドの呼び出しと辺スに格納
number = sum(1,5)    #=> 15

Rubyは最後に評価された式がメソッドの戻り値になるのが特徴ですので、returnは省略可能です。 

def sum(num1, num2)
  result = 0
  (num1..num2).each do |num|
    result += num
  end
  result
end

#メソッドの呼び出しと辺スに格納
number = sum(1,5)    #=> 15

■メソッドの引数について

引数のないメソッドであれば、（）ごと省略してメソッドを定義できます。

def say_hello
  'Hello!'
end

引数がある場合、()を付けてメソッドを定義します。引数があっても（）は省略できます。

def say_hello(name)
  'Hello! #{name}'
end

#メソッドを呼び出す
say_hello('Anne')   #=>"Hello! Anne"

#()は省略できる
def say_hello name
  "Hello! #{name}"
end
say_hello('Anne')    #=> "Hello! Anne"

■デフォルト値付きの引数について

Rubyではメソッドを呼び出すときに引数が過不足があるとエラーが発生します。

def say_hello(name)
  'Hello! #{name}'
end

#引数が足りない
say_hello 
#=> ArgumentError: wrong number of arguments (given 0, expected 1) 

#引数が多い
say_hello('Anne', 'John')    
#=> ArgumentError: wrong number of arguments (given 2, expected 1)

引数にデフォルト値を設定しておくと、引数に不足があってもエラーが発生しません。

def say_hello(name= 'Micheal')
  'Hello! #{name}'
end

#引数が足りない
say_hello #=> "Hello! Micheal"

say_hello('Anne')    #=> "Hello! Anne"
