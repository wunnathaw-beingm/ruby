Method Search Rules

Ruby ရဲ့  searching rules ကို to_s method ကိုသုံးပြီးတော့ ပြပါမယ်. 

module ModuleA
  def to_s
    "ModuleA #{super}"
  end
end

module ModuleB
  def to_s
    "ModuleB #{super}"
  end
end

class ClassA
  def to_s
    "ClassA #{super}"
  end
end

class ClassB < ClassA
  include ModuleA
  include ModuleB
  def to_s
    "ClassB #{super}"
  end
end

obj = ClassB.new
obj.to_s  #=> "ClassB ModuleB ModuleA ClassA #<ClassB: 0x0000000035237c8>"

 ClassB.ancestors  #=> [ClassB, ModuleB, ModuleA, ClassA, Object, Kernel, BasciObject]

အထက်ပါ code ကိုrunကြည့်ရင် to_s method ကို အောက်ပါအတိုင်းခေါ်သွားတာ တွေ့ရမှာပါ
・class B ရဲ့ to_s method
・2 ခုမြောက်မှာ include လုပ်ထားတဲ့ moduleB ရဲ့ to_s method
・1 ခုမြောက်မှာ include လုပ်ထားတဲ့ moduleA ရဲ့ to_s method
・super cass ဖြစ်တဲ့ classA ရဲ့ to_s method
・ClassA ရဲ့ super class ဖြစ်တဲ့ object class ရဲ့ to_s method

ဘယ်လိုအစဥ်လိုက် ခေါ်သုံးသွားလဲဆိုတာကို ancestors method ကိုသုံးပြီး ကြည့်နိုင်ပါတယ်.တကယ်လို့ နောက်ဆုံးဖြစ်တဲ့ BasicObject class မှာသာ method ကို မတွေ့ခဲ့လျင်တော့ NoMethodError ပဲပြမှာဖြစ်ပါတယ်.

Module with State

class လည်း data ကိုသိမ်းလို့ရသလိုမျိုးပဲ module လည်း data ကိုသိမ်းလို့ရပါတယ်. 

module TestApi
    @url = ''
    @debug_mode = false
    
    class << self
      attr_accessor :url, :debug_mode  
    end
end

TestApi.url = "http://example.com"
TestApi.debug_mode = true

puts TestApi.url            #=> "http://example.com"
puts TestApi.debug_mode     #=> true

အထက်ပါပုံစံအတိုင်းကို module နဲ့မရေးဘဲ class နဲ့လည်း ရေးနိုင်ပါတယ်. class နဲ့ module ကွာတဲ့အချက်က class က instance ဆောက်လို့ရတယ် module က instance ဆောက်လို့မရဘူး..ဒါပဲကွာတာပါ.အာ့‌ကြောင့် လိုအပ်ချက်အရ  instance  မဆောက်စေချင်တဲ့အခါကျရင် module ကို အသုံးပြုနိုင်ပါတယ်.

နောက်တစ်နည်းအနေနဲ့ class လိုမျိုး class << self ကိုလည်း သုံးလို့ရပါသေးတယ်. အဲ့သလိုသုံးမယ်ဆိုရင်  Module ထဲမှာ method တွေ အများကြီးရှိရင်တော့  ဒါက အသုံး၀င်ပါတယ်. method တစ်ခုချင်းစီကို self.  ဆိုတာကြီးလိုက်ထည့်နေစရာမလိုတဲ့အတွက်ပါ.

    #ဒီအောက်မှာ ရေးသမျှသည် module_function တွေ ဖြစ်လာမှာပါ.

Namespace

namespace ဆိုတာ နာမည်ထပ်တာမျိုးတွေမဖြစ်အောင် သုံးရခြင်းဖြစ်ပါတယ်. project ကကြီးလာရင် သူများရေးထားတဲ့ class name တွေ function name တွေနဲ့ သွားတူနေတာမျိုးတွေဖြစ်နိုင်ပါတယ်. နာမည်တူ method တွေ class တွေ ရှိလာတဲ့အခါမှာ  ကိုယ်ကခေါ်သုံးချင်တာကတစ်ခုရောက်လာတာကတစ်ခု တစ်ခုနဲ့တစ်ခု မှားခေါ်မိတာတွေ ဖြစ်လာနိုင်ပါတယ်. namespace  ကိုသုံးခြင်းအားဖြင့် အဲ့သလိုတွေ မဖြစ်ရအောင်  ကာကွယ်နိုင်ပါတယ်. ဥပမာ.

Comparable Module

Comparable Module ဆိုတာ compare operator တွေကို အသုံးပြုနိုင်တဲ့ module ဖြစ်ပါတယ်.Comparable Module ကို include လုပ်ထားမယ်ဆိုရင် အောက်ပါ method တွေကို အသုံးပြုနိုင်မှာဖြစ်ပါတယ်.<    <=    ==    >    >=    between?

Comparable Module ကိုသုံးနိုင်ဖို့ဆိုရင် include လုပ်မယ့် class မှာ <=> operator ကို implement လုပ်ပေးထားရမှာပဲဖြစ်ပါတယ်.<=> operator ကို UFO operator လို့လည်း ခေါ်ကြပါသေးတယ်.
a <=> b ဆိုရင အောက်ပါresult 4 ခုကိုပြန်ပေးနိင်တယ်.

  1. a > b ဆိုရင် 1
  2. a == b ဆိုရင် 0
  3. a < b ဆိုရင် -1
  4. နှိုင်းယှဥ်လို့မရရင် nil

Comparable Module ကို include လုပ်ပြီးတော့ သုံးထားတဲ့ပုံစံကို အောက်က ဥပမာကိုကြည့်ပါ. 

class Tempo
    include Comparable

    attr_reader :bpm

    #bpm ဆိုတာ beats per minute ကိုဆိုလိုတာ
    def initialize(bpm)
        @bpm = bpm
    end

    def <=>(other)
        if other.is_a?(Tempo)
            #bpm ကို <=> နဲ့compareလုပ်မယ်
            bpm <=> other.bpm
        else
            #compare မလုပ်နိုင်ရင် nil ကို return ပြန်မယ်
            nil
        end
    end

    def inpect
        "#{bpm}bpm"
    end
end

t_120 = Tempo.new(120)      #=> 120bpm
t_180 = Tempo.new(180)      #=> 120bpm

t_120 > t_180       #=> false
t_120 <= t_180      #=> true
t_120 == t_180      #=> false

Enumerable Module

Enumerable Module ဆိုတာ Array, Hash, Range တို့လို loop ပတ်လို့ရတဲ့ class တွေထဲမှာ include လုပ်ထားတဲ့ module ဖြစ်ပါတယ်.Enumerable Module ကို include လုပ်ထား မထားဆိုတာကို include? method နဲ့ စစ်ကြည့်လို့ရပါတယ်. 

Array.include?(Enumerable)      #=> true
Hash.include?(Enumerable)       #=> true
Range.include?(Enumerable)      #=> true

Enumerable Module မှာဆိုရင် map, select, find, count တို့လို method တွေအများကြီးပါဝင်ပါတယ်.Enumerable Module ကို include လုပ်ထားတဲ့ class ဆိုရင် အဲ့ဒီ method တွေကို အသုံးပြုနိုင်ပါတယ်. 

#Array, hash, range မှာ map method ကို အသုံးပြုထားပုံ
[1, 2, 3].map { |n| n * 10 }                     #=> [10, 20, 30]
{ a: 1, b: 2, c: 3 }.map { |k, v| [k, v * 10] }  #=> [[:a, 10], [:b, 20], [:c, 30]]
(1..3).map { |n| n * 10 }                        #=> [10, 20, 30]
#all method
[1,2,3].all? { |v| v > 0 }      #=> true

[1,2,3].all? { |v| v > 1 }      #=> false
#any method
[1,2,3].any? { |v| v == 2 }     #=> true
# collect method
[1,2,3].collect { |v| v * 2 }   #=> [2, 4, 6]

Module Mixin

class တစ်ခုမှာ module ကို include လုပ်လိုက်တာကို mixin လို့ခေါ်ပါတယ်. 

module Loggable
  def log(text)
    puts "[LOG] #{text}"
  end
end

class Product
  #module ကို include လုပ်မယ်
  include Loggable
  
  def name
    log 'name is called'
    'MOVIE'
  end
end

product = Product.new
product.name  #=> [LOG] name is called
              #=> MOVIE

■prepend ကိုသုံးနည်း

prepend နည်းလမ်းနဲ့ module ကို mixin လုပ်နိုင်ပါတယ်.prepend ရဲ့ feature တစ်ခုကတော့ method name တူတဲ့အခါမျိုးမှာ mixin လုပ်ထားတဲ့ class ထက်module ရဲ့ method ကိုပဲခေါ်သုံးမှာဖြစ်ပါတယ်.

module M
  def foo
    puts "Module: foo"
  end
end

#module ကို prepend လုပ်ထားတဲ့ class
class PrependModule
  prepend M
  def foo
    puts "Class: foo"
  end
end

prependModule = PrependModule.new
prependModule.foo       #=>   "Module: foo"


#module ကို include လုပ်ထားတဲ့ class
class IncludeModule
  include M
  def foo
    puts "Class: foo"
  end
end

includeModule = IncludeModule.new
includeModule.foo      #=> "Class: foo"

■module ထဲမှာ တခြား module ကို include လုပ်ခြင်း

include လုပ်တာဟာ class တစ်ခုတည်းမှာတင်မဟုတ်ပါဘူး. module အချင်းချင်းမှာလည်း  include လုပ်လို့ရပါတယ်. 

module Foo
    def foo
        puts "Foo"
    end
end

module Bar
    #တခြား module ကို include လုပ်
    include Foo
    def bar
        puts "Bar"
    end
end

class C
    #Bar module ကို include လုပ်
    include Bar
end

c = C.new

c.bar       #=>  "Bar"
c.foo       #=>  "Foo"

Defining Module

Ruby ရဲ့ module ဟာ class လိုပဲ module အတွင်းထဲမှာ  method  တွေ application  အတွက်လုပ်ရမယ့် အလုပ်တာ၀န်တွေကို စုစည်းထည့်သွင်းထားနိုင်ပါတယ်. module တစ်ခုကို အောက်ပါအတိုင်း create လုပ်နိုင်ပါတယ်. 

#hello method တစ်ခုကိုdefineလုပ်ထားတဲ့Greeter Module
module Greeter
  def hello
    'hello'
  end
end

အထက်ပါ code ကိုကြည့်မယ်ဆိုရင် class တစ်ခု create လုပ်တာနဲ့ပုံစံအတူတူပဲဖြစ်တာကို တွေ့ရမှာပါ.ဒါပေမယ့် module နဲ့ class နဲ့မတူတာက module က instance ေဆာက်လို့မရတာပဲ ဖြစ်ပါတယ်.နောက်တစ်ခုက တခြား module တွေ class တွေကို  inheritance  လုပ်လို့မရပါဘူး. 

#hello method တစ်ခုကိုdefineလုပ်ထားတဲ့Greeter Module
module Greeter
  def hello
    'hello'
  end
end

#module ဟာ instance ဆောက်လို့မရပါဘူး
greeter = Greeter.new    #=> NoMethodError

#တခြား module ကို inheritance လုပ်ပြီးတော့ module အသစ်တစ်ခု create လုပ်လို့မရပါဘူး
module AwesomeGreeter < Greeter
end

Monkey Patch

monkey patch ဆိုတာက class တွေ library တွေရဲ့ အရင်ရှိနှင့်ပြီးသား method တွေကို  override လုပ်ပြီးကိုယ်လိုချင်တဲ့ပုံစံကို ပြောင်းရေးတာပါ. 

#User class ကိုအပြင် libraryအနေနဲ့defineလုပ်ထား
class User
  def initialize(name)
    @name = name
  end
  
  def hello
    "Hello, #{@name}!"
  end
end

#Monkey Patch မလုပ်ခင် အခြေအနေ
user = User.new('John')
user.hello    #=> "Hello, John!"

#hello method မှာ Monkey Patch နဲ့ process ပုံစံကိုပြောင်းလိုက်ပါတယ်.
class User
  def hello
    "#{@name}、Nice to meet you!"
  end
end

user.hello    #=>  "John, Nice to meet you!"

အရင်ကရှိခဲ့တဲ့ method ကို alias method အနေနဲ့ ချန်ထားခဲ့ပြီးတော့ override လုပ်တဲ့ method ထဲကနေ အရင် method ကိုခေါ်တာမျိုးလည်း လုပ်လို့ရပါတယ်. 

#User class ကိုအပြင် libraryအနေနဲ့defineလုပ်ထား
class User
  def initialize(name)
    @name = name
  end
  
  def hello
    "Hello, #{@name}!"
  end
end

#Monkey Patch
class User
  #အရင်hello method ကို hello_original အနေနဲ့ခေါ်ပြီးသုံးလို့ရအောင်လုပ်
  alias hello_original hello
  
  def hello
    "#{hello_original}, WelCome!"
  end
end

user = User.new('John')
user.hello    #=> "Hello, John!, Welcome!"
