Objects, Instances, Receivers

class ဟာဆိုရင် structure တစ်ခုပါပဲ.car structure ကနေပြီးတော့ အနီရောင်ကား အနက်ရောင်ကားတွေ ထုတ်နိုင်သလို OOP မှာလည်း class ကနေပြီးတော့  object  အမျိုးမျိုးကို  create လုပ်နိုင်ပါတယ်.
တူညီတဲ့ class ကနေ create လုပ်ခံထားရတဲ့  object  တွေမှာ  method တွေ  attribute  တွေဟာတူတူပဲ ရရှိလာမှာပါပဲ. ဒါပေမယ့် အဲ့ဒီ  attribute  တွေထဲမှာရှိနေမယ့်  data တွေကတော့ တူမှာမဟုတ်ပါဘူး.object တစ်ခုချင်းစီအလိုက်ကွဲသွားပါလိမ့်မယ်. 

#Alice Ruby အသက် 20 ဆိုတဲ့ user object တစ်ခု create လုပ်လိုက်တာ
alice = User.new('Alice', 'Ruby', 20)
#Bob Python အသက် 30 ဆိုတဲ့ user object တစ်ခု create လုပ်လိုက်တာ
bob = User.new('Bob', 'Python', 30)

#object 2 ခုစလုံးမှာ full_name ဆိုတဲ့ method ပိုင်ဆိုင်ထားပေမယ့်
#အထဲက data ကတော့ တူညီမှာမဟုတ်ပါဘူး.
alice.full_name     #=> Alice Ruby
bob.full_name       #=> Bob Python

အဲ့ဒီလို class တစ်ခုကို အခြေခံထားပြီးတော့ data တွေ ဆောက်ဆောက်သွားတာကို  object လို့ခေါ်ပါတယ်. အဲ့ဒီလိုပဲ တစ်နည်းအားဖြင့် instance လို့လည်း ခေါ်ကြပါသေးတယ်. အဲ့တာကြောင့် object ဆိုတာနဲ့ instance ဆိုတာနဲ့က အတူတူပဲလို့မှတ်လို့ရပါတယ်။

နောက်တစ်ခု object နဲ့ method နဲ့တွဲလာတဲ့အခါမှာဆိုရင် object ကို receiver လို့ခေါ်ပါတယ်. 

user = User.new('Alice', 'Ruby', 20)
user.full_name

အပေါ်က code မှာ user object ရဲ့ full_name method ကို ခေါ်ထားပါတယ်. အဲ့ဒီမှာ  full_name method ရဲ့ receiver က user ဖြစ်ပါတယ်.

အဲ့သလိုပဲ full_name ကို method လို့ခေါ်ကြသလို တချို့နေရာတွေမှာ  function  တို့  subroutine  လို့လည်းခေါ်ကြပါသေးတယ်.Ruby မှာတော့  function  တို့  subroutine  လို့ ခေါ်လေ့ ခေါ်ထမရှိပါဘူး.

ပြောချင်တာတစ်ခုက message ဆိုတဲ့ အခေါ်အ‌‌ဝေါ်ပါ. user.full_name  မှာ  user ကို  receiver  လို့ခေါ်ပြီးတော့  full_name ကိုတော့ message လို့ခေါ်ကြပါသေးတယ်.

receiver တို့ message တို့ဆိုတဲ့ အခေါ်အဝေါ်ဟာ  Smalltalk  ဆိုတဲ့  language  မှာ  သုံးတဲ့အသုံးအနှုန်းပါ. Ruby လည်း Smalltalk ရဲ့ တစ်ချို့အပိုင်းတွေကို ယူထားတာရှိလို့  receiver  တို့  message  တို့ဆိုတဲ့ အခေါ်အဝေါ်တွေကို သုံးတတ်ပါသေးတယ်.

Super

super ကိုသုံးမယ်ဆိုရင် parent class မှာရှိတဲ့ name တူတဲ့ method ကို  ခေါ်သွားမှာဖြစ်ပါတယ်။
အောက်ပါ ဥပမာမှာ ဆိုရင် parent ရဲ့ initialize method ကို super ကိုသုံးပြီး ခေါ်သွားပါတယ်။
parent ရဲ့ initialize method မှာ parameter value တွေပါလို့ super ကနေလဲ parameter တွေပေးပြီးခေါ်ရပါတယ်။ 

class Parent
  attr_reader :name, :age
  
  def initialize(name, age)
    @name = name
    @age = age
  end
end

class Child < Parent
  attr_reader :grade
  
  def initialize(name, age, grade)
    super(name, age)
    @grade = grade
  end
end

john = Child.new("John", 18, 10)
john.name  #=> "John"
john.age   #=> 18
john.grade #=> 10

super class နဲ့ subclass ရဲ့ parameter ကအတူတူဖြစ်တဲ့အချိန်မှာ parameter ထည့်ပေးစရာမလိုပါဘူး။ super လို့ရေးလိုက်ရုံနဲ့ parameter passing ဖြစ်သွားပါမယ်။

super class နဲ့ subclass ရဲ့  လုပ်ဆောင်ပုံခြင်း က တူတူပဲဆိုရင်  subclass မှာ ဒါတွေကို မလုပ်လဲ ရပါတယ်။
・method name တူတဲ့ method ကို create လုပ်တာမျိုး
・super ကို လှမ်းခေါ်တာမျိုး 

class Parent
  attr_reader :name, :age
  
  def initialize(name, age)
    @name = name
    @age = age
  end
end

# super နဲ့ subclass က paremeter အတူတူပဲဆိုရင်
class Child < Parent
  def initialize(name, age)
    super
  end
end

john = Child.new("John", 18)
john.name  #=> "John"
john.age   #=> 18

# process မကွဲဘူးဆိုရင် same method create လုပ်တာ
# super ကိုခေါ်တာ မလုပ်လဲ ရပါတယ်
class Child < Parent
end

john = Child.new("John", 18)
john.name  #=> "John"
john.age   #=> 18

initialize method နဲ့ super သုံးပုံသုံးနည်းကို ဥပမာပြခဲ့ပေမယ့် 
initialize method အပြင် တခြား method တွေမှာလည်း တူတူပဲ သုံးနိုင်ပါတယ်.

Inheritance in Class

Ruby မှာ class တစ်ခုကို define လုပ်လိုက်ရင် object class ကနေ inheritance လုပ်ပြီးသားဖြစ်သွားပါတယ်.object class  မဟုတ်တဲ့တခြား class ကို inheritance လုပ်မယ်ဆိုရင် < လေးကိုသုံးပါတယ်.object class ကနေ inheritance လုပ်ထားတဲ့ဟာမျိုးဆို < လေးထည့်စရာမလိုပါဘူး.
ဥပမာ  ..

class Product
end

ဆိုရင် ပုံမှန်ပဲ Product class တစ်ခုကို create လုပ်ထားတာပါ. create လုပ်လိုက်တာနဲ့  တစ်ပြိုင်နက် object class ကနေ inheritance လုပ်ပြီးသားလည်းဖြစ်သွားပါတယ်. object class ကနေ inheritance လုပ်တယ်ဆိုပေမယ့် < လေးကို ထည့်စရာမလိုပါဘူး..

object class မဟုတ်တဲ့တခြား class ကနေ inheritance လုပ်မယ်ဆိုရင် အောက်ပါအတိုင်း ဖြစ်ပါမယ် 

class Product
end

#DVD class က Product class ကို inheritance လုပ်ထားတာ
class DVD < Product
end

အဲ့ဒီလို parent/child relationship ဖြစ်နေတဲ့အချိန်မှာ ခေါ်ပုံခေါ်နည်းတွေရှိပါတယ်. Product class ကို Super class လို့ခေါ်ပြီးတော့ DVD class ကိုတော့ sub class လို့ ခေါ်ပါတယ်.

object ရဲ့ class ကို ရှာချင်ရင် class method ကို သုံးပြီးရှာလို့ရပါတယ်. 

user = User.new
user.class      #=> User

နောက်တစ်နည်းအနေနဲ့ instance_of? method ကိုသုံးပြီးလည်း ရှာနိုင်ပါတယ်. 

user = User.new

#user က User Class ရဲ့ instance လား
user.instance_of?(User)     #=> true

#user က String Class ရဲ့ instance လား
user.instance_of?(String)     #=> false

Inheritance relationship(is-a relationship) ကို ရှာချင်ရင် is_a? method ကိုသုံးပြီးတော့ရှာနိုင်ပါတယ်. alias method အနေနဲ့ kind_of? method ကိုသုံးနိုင်ပါတယ်. 

user = User.new

#instance_of? method က instance တိုက်ရိုက်မဟုတ်ရင် false ပဲပြန်ပါတယ်.
user.instance_of?(Object)   #=> false

#is_a? method ကတော့ is-a relationship ဖြစ်တာနဲ့ true ကိုပြန်ပေးပါတယ်.
user.is_a?(User)            #=> true
user.is_a?(object)          #=> true
user.is_a?(BasicObject)     #=> true

#is-a relationship မဟုတ်ရင် false
user.is_a?(String)        #=> false

Self Keyword

Ruby ရဲ့ self က instance ကိုယ်တိုင်ကို ဖော်ပြတဲ့ keyword ပဲဖြစ်ပါတယ်. class ထဲမှာ  self ကိုရေးထားရင် အဲ့ဒီ class ရဲ့ instance variable ကို reference လုပ်တာပါ. 

class User
  attr_accessor :name
  
  def initialize(name)
    self.name = name
  end
  
  def my_name 
    "my name is #{name}."
  end
  
  def greet
    "Hello, I'm  #{self.name}"
  end
  
end

john = User.new("John")
john.my_name    #=> "my name is John"
john.greet      #=> "Hello, I'm John"

method မှာသာ self ကိုထည့်သုံးမယ်ဆိုရင် အဲ့ဒီ  method  ဟာ  instance  method  မဟုတ်တော့ဘဲ  class method ဖြစ်သွားမှာပါ 

class User
  attr_accessor :name
  
  def initialize(name)
    self.name = name
  end
  
  def self.my_name
    "This is class Method"
  end
  
  def my_name
    "My name is #{name}"
  end
end

john = User.new("John")
john.my_name    #=> "My name is John"
User.my_name    #=> "This is class Method"

တစ်ချို့နေရာတွေမှာ self မထည့်မိလို့ error တက်တာမျိုးတွေလည်းဖြစ်နိုင်တာမို့လို့  self ကိုသုံးတဲ့အခါမှာ သတိထားကြပါ. 

class User
  attr_accessor :name
  
  def initialize(name)
    @name = name
  end
  
  def rename_to_John
    self.name = "John"
  end
  
  def rename_to_Micheal
    #self ထည့်ဖို့ကိုမေ့ကျန်ခဲ့တာ
    name = "Micheal"

foo = User.new("Foo")

Public Level of Class Method

Ruby ရဲ့ method မှာ Public Level 3 ခုရှိပါတယ်.

  • public
  • protected
  • private

■public method

public method ဟာဆိုရင် class အပြင်ကနေ လှမ်းခေါ်လို့ရတဲ့ method ဖြစ်ပါတယ်. initialize method ကလွဲရင် default အနေနဲ့ public ဖြစ်ပါတယ်. 

class User
  def greeting
    'Hello!'
  end
end

user = User.new
user.greeting  #=> 'Hello!'

■private method

private method ဟာ class အပြင်ကနေ ခေါ်သုံးလို့မရပါဘူး. class အတွင်းထဲမှာပဲ ခေါ်သုံးခွင့်ရှိပါတယ်. class အတွင်းထဲမှာ private keyword ကိုရေးထားလိုက်ရင် အဲ့ဒီ  private  keyword  အောက်ပိုင်းမှာ define လုပ်ထားသမျှ method တွေဟာ  private method တွေဖြစ်လာပါလိမ့်မယ်. 

class User
  #ဒီအောက်ပိုင်းမှာ define လုပ်သမျှ method တွေဟာ private method ဖြစ်လာမှာပါ
  private
  
  def greeting
    'Hello!'
  end
end

user = User.new
user.greeting  #=> NoMethodError

private method ဟာ receiver နဲ့ခေါ်သုံးမယ်ဆိုရင်တောင် ခေါ်သုံးလို့မရတဲ့  method  ဖြစ်ပါတယ်. 

class User
  #receiver self ကိုသုံးပြီးခေါ်ပေမယ့် name method က private method မို့လို့ error တက်ပါလိမ့်မယ်
  def greeting
    "Hello, I am #{self.name}."
  end
  
  private
  
  def name
    'John'
  end
end

user = User.new
user.greeting   #=> NoMethodError: private method 'name' called for <User>

private method ဟာ subclass ကနေ ခေါ်သုံးလို့ရနိုင်ပါတယ်. 

class Animal
  private
  def species
    "Animal Species"
  end
end

class Dog < Animal
	def type
      #species က super class ရဲ့ private method
      "type: #{species}"
    end
end

pappy = Dog.new
pappy.type   #=> "type: Animal Species"

super class ရဲ့ private method ကို ခေါ်သုံးနိုင်လို့ override ကိုလည်းလုပ်နိုင်ပါတယ်. 

class Animal
   
  def animalType
    "type: #{species}"
  end
  
  private

■Local variable

local variable အကြောင်းကို အရင် chapter တွေမှာ ပြောထားခဲ့ပြီဖြစ်လို့ ဒီနေရာမှာ မပြောတော့ပါဘူး

■Instance variable

class အတွင်းထဲမှာ instance variable ကိုသုံးနိုင်ပါတယ်. instance variable ကို တူညီတဲ့ object, တူညီတဲ့ instance ကနေ access လုပ်ပြီးခေါ်သုံးနိုင်ပါတယ်. 

class User
    def initialize(name)
        #instance variable ထဲကို parameter value ထည့်လိုက်ပါမယ်
        @name = name
    end

    def hello
        #instance variable ထဲက data ကို ထုတ်ပြပါမယ်
        "Hello, I am #{@name}."
    end
end

user = User.new('Alice')
user.hello     #=> "Hello, I am Alice."

■Class instance variable

instance variable ဆိုတာ @ နဲ့စပါတယ်.class instance variable ဆိုတာကတော့ class အတွင်းထဲမှာ ရေးထားတဲ့ variable ကိုဆိုလိုပါတယ်.method ရဲ့အတွင်းမှာ ရေးထားတဲ့ variable မဟုတ်ပါဘူး.method ရဲ့အတွင်းမှာ ရေးထားတဲ့ variable ဆိုရင် instance variable ဖြစ်သွားပါမယ်.
အောက်ပါ ဥပမာကို ကြည့်ပါ 

class Product
    #class instance variable
    @name = 'Product'

    def self.name
        #class instance variable
        @name
    end

    def initialize(name)
        #instance variable
        @name = name
    end

    def name
        #instance variable
        @name
    end
end

product = Product.new('movie')
product.name        #=> movie
Product.name        #=> Product

class instance variable ဖြစ်တဲ့အတွက် instance ဆောက်စရာမလိုဘဲ  class  ကနေ  Product.name ဆိုပြီးတော့ တိုက်ရိုက်ခေါ်သုံးလို့ရပါတယ်.

■Class variable

class variable ဟာ class အတွင်းထဲမှာ, class ရဲ့ method အတွင်းထဲမှာ,  super  class,  subclass တွေကနေလည်း ခေါ်သုံးနိုင်ပါတယ်.တစ်နေရာကနေ update လုပ်လိုက်ရင်  အကုန်လုံးလိုက်ပြောင်းသွားမှာဖြစ်ပါတယ်. class variable ဟာ @@ နဲ့စပါတယ်.

■Global variable

Ruby မှာ global variable ဆိုပြီးတော့ variable တစ်မျိုးရှိပါသေးတယ်. global variable ဟာဆိုရင် $ နဲ့စပါတယ်.global variable ဟာဆိုရင် class အတွင်းတွေ အပြင်တွေတင်မဟုတ်ဘဲ program ကြီးတစ်ခုလုံးရဲ့ဘယ်နေရာကမဆို ခေါ်သုံးလို့ရပါတယ်.

■Constant

Regular Expression ကို case နဲ့တွဲပြီး သုံးနိုင်ပါတယ်.

text = '03-1234-5678'

case text
  when /^\d{3}-\d{4}$/
  	puts 'post number'
  when /^\d{4}\/\d{1,2}\/d{1,2}$/
  	puts 'date'
  when /^\d+-\d+-\d+$/
    puts 'phone number'
end

#=>  phone number
